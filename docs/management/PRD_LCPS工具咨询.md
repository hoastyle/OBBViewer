2. 背景:
   1. LCPS是系统中极其重要的安全功能，且如果有漏报，会对安全有重大影响。如果有频繁误报，则会对效率产生重大影响。
   2. LCPS是基于激光的功能，涉及到大量的点云操作。
   3. LCPS到目前为止，经常性出现的问题如下:
      1. 逻辑复杂，少量改动可能导致LCPS功能出现异常，在需要报警时漏报。（且无法提前预期、无法感知，只有碰撞后才能通过目前已有的少量记录功能再现当时的场景）。总结来说，整体逻辑修改后出现异常，引发安全风险
      2. 基于1，还有可能出现LCPS未按预期开启、关闭等等生命周期问题，引发安全风险。
      3. 由于激光安装方案的不同（激光数目不同，种类不同），堆场环境的不同（高低不同，则距离激光的距离不同），会导致堆场中不同的区域稀疏度不痛。在经过过滤等操作后，原来可以正常声场障碍物的场景，往往可能会无法生成预期的障碍物。总结来说，就是障碍物生成不及预期。
      4. 出现问题后，难以定位。必须研发同事获取过程中录制的中间数据并通过该数据恢复场景（有些漏报场景数据会缺失，导致问题难以复现和定位）
   4. 点云导致数据传输量大，难以使用工具实时传输、观测等。
3. 需求
   1. 能够方便、快捷的观测LCPS（更多的信息、观测数据（点云、OBB、状态、图像）等等）
   2. 实时监控LCPS的防护情况，能够及时确认LCPS是否有漏报、误报问题（当前的recvOBB.py主要就是实现这部分功能）
   3. 高级功能: 方便、快捷的对LCPS的问题进行定位、调试
   4. 整个观测、调试过程中的数据能记录，可回溯。 
   5. 不能够影响现场系统的运行，特别是影响LCPS自身功能的运行。（比如，LCPS发送OBB的数据都是非阻塞性发送、且仅发送压缩后的OBB数据，不包含原始点云数据等等）
